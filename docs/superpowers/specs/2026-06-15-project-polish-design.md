## 三、P1 — 错误处理增强

### 3.1 当前问题

- `fetch()` 使用裸 `try/except`，失败后静默返回空字符串
- 无重试机制 — 网络抖动直接丢数据
- 无超时区分 — 连接超时和读取超时未分开处理
- DeepSeek API 调用仅一次尝试，失败即跳过整篇文章
- 无降级方案 — API 不可用时没有任何缓存或回退

### 3.2 设计方案

**data_collector.py：**

```python
def fetch_with_retry(url, timeout=10, retries=2):
    """带重试的 HTTP 请求，指数退避"""
    for attempt in range(retries + 1):
        try:
            r = requests.get(url, headers=HEADERS, timeout=timeout)
            r.encoding = r.apparent_encoding
            if r.status_code == 200:
                return r.text
        except requests.Timeout:
            print(f"  [RETRY] 超时({attempt+1}/{retries})")
        except requests.ConnectionError:
            print(f"  [RETRY] 连接失败({attempt+1}/{retries})")
        time.sleep(2 ** attempt)  # 指数退避
    return ""
```

**analyzer.py：**

- `call_deepseek()` 添加重试（2 次 + 指数退避）
- 添加 API 健康检查：连续 3 次失败进入"降级模式"（用本地规则分析替代）
- 降级模式：基于关键词的简单分类 + 中性情绪默认值

### 3.3 实现范围

| 文件 | 改动 |
|------|------|
| `data_collector.py` | 新增 `fetch_with_retry()`，改造 `fetch()` → 包装重试，`fetch_json()` 同理 |
| `analyzer.py` | `call_deepseek()` 加重试 + 降级开关 |
| `config.py` | 新增 `FETCH_RETRIES`, `API_RETRIES`, `API_TIMEOUT` 等配置项 |

## 四、P2 — AI 洞察写作优化

### 4.1 当前问题

- 洞察 prompt 写出的文本偏"通用感" — 数据罗列多于政策分析深度
- 缺乏专业术语密度，不够像真正的财政政策研究报告
- 标题和分段结构单一

### 4.2 设计方案

优化 `generate_weekly_insight()` 中的 prompt，增加：

- 角色设定增强 → "国务院发展研究中心研究员"
- 要求引用具体数据源（如"根据财政部本周发布的XX文件"）
- 结构模板化 → 三段式：总体判断 → 重点领域 → 趋势展望
- 风格要求 → "语言精炼，有政策分析深度，避免空泛描述"
- 要求输出带 Markdown 结构（**重点加粗**、关键数据高亮）

## 五、P3 — 封面图 AI 生成

### 5.1 当前问题

- 封面图是 Pillow 文字渲染，设计感弱
- 纯色背景 + 文字，类似 PPT 默认风格

### 5.2 设计方案

用 DeepSeek API 生成封面图（需确认 DeepSeek 是否支持图像生成）。

**备选方案：**

| 方案 | 方式 | 可行性 |
|------|------|--------|
| A. DeepSeek 图像生成 | 调用 DeepSeek 图像 API | 待确认（可能不支持） |
| B. 本地 Pillow 增强 | 渐变背景 + 随机几何装饰 + 更好的字体组合 | 无需额外依赖，确定可行 |
| C. 文字描述生成 SVG | 用 Python 生成 SVG 封面 | 纯文本方案，嵌入浏览器显示效果佳 |

**推荐方案 B**（Pillow 增强）作为保底，若 DeepSeek 支持图像则用方案 A。
